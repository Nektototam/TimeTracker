# Управление проектом (Boards & Tasks)

Цель: обеспечить прозрачность работ, предсказуемость релизов и устойчивое качество.

## 1) Доски (Boards)

Рекомендуемая структура досок (Kanban):

- Backlog
- Ready
- In Progress
- Review
- Done

Правила:

- Любая задача проходит через Review.
- В In Progress не более 2 задач на человека.
- Оценка сложности и приоритета обязательна перед Ready.

## 2) Задачи (Tasks)

Каждая задача содержит:

- Четкое описание результата
- Критерии приемки (Definition of Done)
- Оценку (S/M/L или Story Points)
- Ссылку на дизайн/скриншот/контекст (если нужно)

Пример формата:

- Title: Implement /auth/login
- Description: Эндпоинт логина с проверкой пароля
- Acceptance: Возвращает access/refresh, покрыто тестами
- Estimate: M

## 3) Релизы

- Версионирование: SemVer (v1.0.0)
- Минимум 1 стабильный релиз в месяц
- Чеклист релиза: тесты, линт, миграции, smoke-тест

## 5) bd (beads)

Основные команды:

- bd ready
- bd show <id>
- bd update <id> --status in_progress
- bd close <id>
- bd sync

- Все изменения в БД — только через миграции.
- Все изменения API — с версионированием контракта.
- Запрет на прямой доступ frontend к БД.
